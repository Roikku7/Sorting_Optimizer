WHAT IT IS
----------
Electron desktop app that reads a Summoners War account export
(JSON produced by SWEX) and scores every rune to tell you which
ones are junk (sellable) and which are worth keeping.


THE FILES THAT MATTER
---------------------
main.js               Electron MAIN process. Opens the window,
                      owns the 4 IPC handlers (file picker, run
                      analysis, load results, icon paths).
preload.js            Security bridge. Exposes only those 4 IPC
                      calls to the page as window.electronAPI.*
src/logic_analyze.js  THE BRAIN. Pure Node/CommonJS. Reads the
                      SWEX JSON, and for each rune reverse-
                      engineers how many upgrade rolls each substat
                      got ("detectProcsMethodA"), then computes
                      "missPoints" = how far the rune is below its
                      max potential. missPoints > threshold => junk.
                      Also flags "reap" runes (good legendary
                      innates on the right sets/slots).
src/la.obf.js         OBFUSCATED copy of logic_analyze.js. This is
                      what ships in the installer (npm run obfuscate
                      regenerates it). If you edit logic_analyze.js,
                      re-run the obfuscate script or the packaged
                      app won't see your change.
src/ui.jsx            React UI (renderer). File-pick button,
                      filters (slot/set/quality/mainstat/substat),
                      rune grid, detail modal with comparison.
src/logic_rune.jsx    Filter / sort / "compare this rune to your
                      other runes of same set+slot+mainstat" logic
                      used by the UI.
mapping.js            Big lookup table: set IDs -> names, stat type
                      IDs -> names, grades, etc. Single source of
                      truth for game data.
index.html            Vite entry page, loads src/ui.jsx.
assets/icons/*.png    Rune set icons used by the packaged app.
icone/*.png           Same icons, served by Vite for browser-only dev.
assets/icone.ico      App/installer icon.


HOW DATA FLOWS
--------------
1. You click "Sélectionner un fichier JSON"
   -> preload -> main.js 'select-json-file' -> native file dialog.
2. main.js 'run-analysis' calls analyzeRunesFromFile(file)
   (from la.obf.js) and writes the result to
   %APPDATA%/sorting_optimizer/sorting_rune.json
3. The UI loads that result ('load-sorted-runes') and renders the
   grid. All filtering/sorting after that is pure UI, no IPC.

Key scoring ideas:
  - extra = rune quality: 1..5 normal, 11..15 ancient (5/15 = legend)
  - missPoints = sum over substats of (max value it could have had
    given its detected rolls) - (actual value), en excluant les substats
    INUTILES pour le set (pas de double pénalité avec le waste)
  - threshold = 8, +2 or +3 tolerance for good sets
    (Violent/Will/Despair/Destroy/Nemesis)
  - wastePoints = procs investis dans des substats inutiles pour le set
    (table SET_RELEVANCE dans mapping.js, surchargeable via le panneau
    "Réglages", persistée dans %APPDATA%/sorting_optimizer/settings.json)
  - score = substats normalisées (valeur / roll max) pondérées par la
    pertinence du set (CLÉ 1.25 / NEUTRE 1.0 / INUTILE 0), innate à 50%
  - verdicts : JUNK (<+12, mauvais rolls/gaspillage) / A_MONTER (<+12,
    prometteuse : monter avant de juger) / KEEP (>=+12, top N de son
    groupe set+slot[+mainstat pour slots 2/4/6]) / SELL (>=+12, N runes
    font mieux dans le groupe)
  - exceptions qui protègent une rune (badge violet, état objectif
    toujours affiché) : SPD >= 20 en substat (réglable), reap (légendaire
    bon innate, protège du JUNK seulement), quad roll = 4 procs dans une
    même substat -> utilisable en "broken set"
  - toJunk = missPoints + wastePoints > threshold (runes <+12)


HOW TO RUN IN DEV (2 terminals)
-------------------------------
  Terminal 1:   npm run dev        -> Vite dev server on :5173
  Terminal 2:   npm start          -> Electron, loads localhost:5173

  (Order matters: Vite first, then Electron.)
  Test file to load: Roikku7-7294441.json (at project root).

  Dev now loads src/logic_analyze.js directly (not the obfuscated
  la.obf.js), so edits to the analysis logic show up on the next
  "npm start" — no need to re-run obfuscate during development.
  (Obfuscate is only run as part of "npm run dist".)

  Run the unit tests any time with:  npm test  (vitest, 9 tests).


HOW TO COMPILE THE ELECTRON APP (the installer)
-----------------------------------------------
  ONE COMMAND:

      npm run dist

  which does 3 things in a row:
    1. npm run obfuscate   -> regenerates src/la.obf.js
    2. vite build          -> bundles the UI into dist/
    3. electron-builder    -> packages everything into a Windows
                              installer

  OUTPUT: "Account Optimizer Setup 1.0.0.exe" in the release/ folder.
    - release/  = the installer + win-unpacked/ (electron-builder)
    - dist/     = the Vite renderer bundle only (fed into the build)
  A portable unpacked build sits in release/win-unpacked/ — run
  "Account Optimizer.exe" there to test without installing.

  If the build fails, first try: delete dist/ and release/ and
  re-run npm run dist.
